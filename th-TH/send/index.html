FAQ — Send Red Packets
  
  
  
 
1) Getting Started
 
 
 
How do I start?
Add the Moonpacket bot to your Telegram group or channel and grant admin permission.
 
What should I prepare before sending?
Deposit the asset you plan to distribute (for example, top up 10 USDT for testing).
 
Where can I send to?
In 'Send target', select your linked groups/channels. You can send to one or multiple at once.
 
Tip: Red packets work great as an icebreaker. Reply under the packet post to announce the game/theme you want to run.
 
 
 
  
 
 
 
 
  
 
2) Sending
 
 
金額分配有哪些模式？
隨機（拼手氣） 或 固定金額（平均分）。
 
可以一次性或分時段發送嗎？
可以。你可設定一次性全發，或在指定期間內分批釋出。
 
餘額不足會怎麼樣？
系統以你的可用餘額為準。餘額不足時會發送失敗並提示補足。
 
公平性如何保障？
分配在伺服器端執行並留存稽核記錄。
 
 
  
 
3) Technical
 
 
可以要求加入群/關注頻道才可領嗎？
可以。啟用內建條件「必須加入某群／關注某頻」。
 
 
什麼是通關密語？適合哪些場景？
玩家需在外部渠道取得密語，回到頁面正確輸入才可領取。常見做法：
 
     
  • YouTube 長影片中拆分 6 位數字密語，鼓勵完整觀看。
    •  
  • X.com（Twitter） 貼文或 Threads 暗示密語，提升互動。
    •  
  • 直播 僅在直播口播密語，引導到場。
    •  
  • 私密/付費群 專屬密語，保障福利只給核心成員。
    •  
 
 
 
可以用自己的 API 決定誰有資格嗎？
可以。Moonpacket 會在用戶領取前，呼叫你提供的只讀 API 進行校驗。後台只需設定 api_url、api_key（可選擇設定允許的網域）。
 
條件宣告（無 user_id）
 
GET [api_url]?api_key=[api_key]
 
回傳：
 
{
  "data": {
    "level": { "gt": 99 },
    "status": { "eq": "active" },
    "is_blacklist": { "eq": false }
  }
}
 
運算子：eq（可比對任何型別）、gt/lt/gte/lte（需數字）
 
使用者查詢（含 user_id）
 
GET [api_url]?api_key=[api_key]&user_id=[user_id]
 
回傳：
 
{
  "data": {
    "level": 100,
    "status": "active",
    "is_blacklist": false
  }
}
 
Moonpacket 會以「條件宣告」作為門檻，對照「使用者查詢」的實際值；符合即通過，不符則不可領。
 
安全建議
使用只讀端點、啟用 api_key 與允許網域限制；若需跨網域建議以 Cloudflare Workers 中轉與限流。
 
 
 
  
 
4) 資產、曝光與結算
 
 
主流幣與 meme 幣的曝光差異？
主流幣（USDT/TON 等）除了在群內展示，還會出現在首頁瀑布流以提高曝光；meme 幣僅在你的群內展示。
 
 
如何防止作弊與批量機器？
 
     
  • 行為頻率與設備/IP 檢測
    •  
  • 帳號與錢包唯一綁定
    •  
  • 提領驗證碼
    •  
  • AI 風控初審；必要時第三方生物認證 + 人臉 + 證件
    •  
 
 
到哪裡看我發送的紀錄與領取人數？
請到你的個人首頁查看「我發送的紅包」，可見每場的領取人數、剩餘份數與消耗金額；若已啟用，亦可下載報表。
